# Agent Instructions

## Language Policy

- Use English across the whole project by default.
- Write source code, comments, documentation, commit messages, changelogs, issue text, examples, test names, and configuration descriptions in English.
- Do not introduce non-English project content unless there is a specific reason.
- Exceptions are allowed only when:
  - The user explicitly requests a non-English output.
  - A quoted external source, API response, game term, or user-facing string must preserve its original language.
  - A localization file or translation-related change requires non-English text.
- When in doubt, use English.

## Scope

These instructions apply to the entire repository unless a more specific `AGENTS.md` file exists in a subdirectory.
